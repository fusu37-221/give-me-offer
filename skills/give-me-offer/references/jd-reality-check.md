# JD Reality Check

Use this file when the user provides a job description, especially for vague, inflated, or trend-heavy roles.

## Purpose

Some job descriptions are clear; others are packaging. Before tailoring a resume, identify what the job actually is and whether the candidate should apply.

## JD Classification

Classify the real role:

- software engineering
- frontend/backend/full-stack
- AI application / AI agent implementation
- algorithm / ML research
- product manager / product assistant
- operations / content / user growth
- sales / pre-sales / customer success
- implementation / technical support
- data analyst / BI
- administrative / HR / finance
- hybrid or unclear

## Red Flags

### Inflated Junior Role

Signs:

- asks for too many unrelated tools
- entry-level title but senior ownership
- "independently responsible for full lifecycle" for a fresh graduate
- vague "fast growth" with no clear work

Action:

- warn the user
- tailor conservatively
- suggest safer adjacent roles

### AI-Washed Role

Signs:

- title says AI/Agent but tasks are content operation, sales, tagging, or customer service
- no mention of API, workflow, data, evaluation, product integration, or model usage
- requires "AI passion" but not technical deliverables

Action:

- identify the underlying role
- decide whether to position as AI operations, AI product, or AI application
- do not overfit the resume to technical AI if the work is not technical

### Sales Disguised as Product/Operations

Signs:

- heavy mention of customer acquisition, conversion, targets, leads, monetization
- little product requirement or user research work
- compensation heavily performance-based

Action:

- tell the user the role likely has sales pressure
- tailor only if the user accepts that direction

### Overly Vague Internship

Signs:

- "complete tasks assigned by leader"
- no clear department or work output
- broad skill list

Action:

- ask for company/team context if possible
- produce a flexible but not over-specialized resume

## Apply / Maybe / Avoid

Return:

```text
Apply: strong fit and reasonable requirements.
Maybe: possible but needs proof or clarification.
Avoid: mismatch, high risk, or misleading role.
```

## Recruiter Questions

Useful questions:

- 这个岗位日常主要产出是什么？
- 团队现在最需要解决的问题是什么？
- 岗位更偏技术实现、产品设计、运营执行，还是客户转化？
- 对应届/转岗候选人的培养方式是什么？
- 试用期/实习期的考核指标是什么？
