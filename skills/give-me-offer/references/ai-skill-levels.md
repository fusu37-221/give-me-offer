# AI Skill Levels

Use this file when a candidate wants to mention AI, AIGC, LLM, RAG, AI agent, prompt engineering, automation, or AI product experience.

## Purpose

AI-related wording is attractive but easy to overclaim. Classify the candidate's actual ability before writing AI claims.

## Levels

### L1: AI Tool User

Can use AI tools for writing, search, translation, code assistance, or productivity.

Safe wording:

```text
Used AI tools to support research, document drafting, and workflow efficiency under manual review.
```

Avoid:

- AI engineer
- AI agent developer
- model optimization

### L2: AI Workflow Adopter

Can integrate AI tools into a repeatable work process, such as content drafting, customer reply templates, report generation, or research summarization.

Safe wording:

```text
Designed repeatable AI-assisted workflows for [task], including prompt templates, human review, and output refinement.
```

### L3: AI Application Prototype Builder

Can build a small application or prototype using API calls, prompt templates, retrieval, tools, or automation.

Safe wording:

```text
Built a prototype AI application for [scenario], covering input processing, prompt/template design, tool/API interaction, and result review examples.
```

### L4: Production AI Application Developer

Can deliver AI features in a real product or internal system with reliability, logging, evaluation, permissions, and user feedback.

Safe wording:

```text
Implemented AI-assisted [feature/workflow] in [system], including API integration, permission control, logging, evaluation samples, and user feedback handling.
```

### L5: Model / Platform / Research Specialist

Can work on model training, fine-tuning, evaluation infrastructure, ML systems, or research-level contributions.

Safe wording depends on real evidence:

```text
Conducted model evaluation/fine-tuning/serving work for [task], using [method], with [verified metric or artifact].
```

## Classification Questions

Ask:

- Did you only use AI tools, or did you build something with an API/framework?
- Did your work run as a repeatable process?
- Did other users use it?
- Did you evaluate quality or failure cases?
- Did you handle data privacy, permissions, or logging?
- Can you show code, screenshots, workflow docs, or examples?

## Resume Risk Rules

- Do not call L1 or L2 candidates "AI Agent engineers".
- Do not claim RAG, fine-tuning, evaluation, or production deployment unless real.
- For prototypes, clearly use "prototype", "demo", or "personal project".
- Pair AI claims with concrete scenario and artifact.
- Include limitations for AI portfolio projects.

## Role Matching

```text
AI product/operations: L1-L2 may be acceptable if domain and workflow sense are strong.
AI application intern/junior: L3 is preferred.
AI agent engineer: L3 with strong software foundation, or L4.
ML/algorithm engineer: L5 or strong academic evidence.
```
