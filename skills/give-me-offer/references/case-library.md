# Case Library

All cases are fictional. Use them to guide structure, tone, and risk handling.

## Case 1: Undergraduate With No Internship, Marketing to Product Assistant

Candidate:

- marketing major
- no formal internship
- ran a campus account
- organized student club events
- wrote course research reports

Positioning:

```text
Entry-level product/operations candidate with user research, content operation, event execution, and structured writing experience.
```

Strongest assets:

- campus account operation
- course research report
- event planning and feedback collection

Resume strategy:

- put projects and campus operation before awards if awards are weak
- avoid claiming product manager experience
- add one PRD/competitive analysis portfolio project

Example bullets:

```text
- Operated a campus content account by planning weekly topics, drafting posts, collecting user feedback, and summarizing engagement patterns for later content selection.
- Supported student event operations, including registration, participant communication, on-site coordination, and post-event feedback summary.
- Completed a course research report on user consumption behavior, covering questionnaire design, data summary, competitor comparison, and presentation delivery.
```

Gap plan:

- write one PRD for a familiar app feature
- produce one user journey map
- prepare interview explanation for cross-major transition

## Case 2: Full-Stack Intern to Frontend

Candidate:

- computer science student
- full-stack internship
- Vue, Spring Boot, MySQL
- wants frontend role

Positioning:

```text
Frontend-focused developer with full-stack collaboration experience and hands-on Vue project delivery.
```

Resume strategy:

- foreground Vue pages, forms, routing, API integration, state management, and UI details
- keep backend work as collaboration context
- do not claim frontend performance optimization unless real

Example bullets:

```text
- Developed Vue-based admin pages for user, order, and content modules, including routing, form validation, table views, and loading/error states.
- Integrated frontend pages with Spring Boot APIs, coordinated field definitions and error handling, and completed end-to-end module testing.
- Extracted repeated table and dialog patterns into reusable page-level components to improve implementation consistency across management modules. [confirm ownership]
```

Gap plan:

- polish one project frontend and add screenshots
- document component structure
- prepare CSS/layout and browser debugging examples

## Case 3: Full-Stack Developer to AI Agent Application

Candidate:

- 1-3 years full-stack work
- familiar with business systems
- wants AI Agent role
- has limited LLM project experience

Positioning:

```text
Application developer transitioning into AI workflow automation, with experience in business systems, API orchestration, and end-to-end process implementation.
```

Resume strategy:

- do not claim mature AI Agent experience without proof
- translate full-stack workflow experience into tool orchestration and business automation readiness
- add a small agent portfolio project

Example bullets:

```text
- Built full-stack business modules involving user input, permission control, database records, and API orchestration, providing a foundation for AI workflow automation scenarios.
- Mapped multi-step business processes into system interactions across frontend forms, backend services, and persistent records.
- Created a prototype document-assistant workflow using file input, prompt templates, retrieval logic, and answer review examples. [use only if real]
```

One-week portfolio:

- document QA agent
- resume screening helper
- customer service triage assistant
- include tool list, architecture, examples, limitations, and evaluation notes

## Case 4: Graduate Research to Data Analyst

Candidate:

- graduate student
- research data processing
- thesis writing
- wants data analyst

Positioning:

```text
Research-trained analyst with data cleaning, structured problem solving, and technical reporting experience.
```

Resume strategy:

- translate research workflow into data workflow
- foreground tools, datasets, methods, and reproducible analysis
- avoid claiming business analytics unless a business case exists

Example bullets:

```text
- Processed experimental/research data through cleaning, coding, descriptive analysis, and visualization to support thesis conclusions.
- Built reproducible analysis records, including data dictionary, intermediate outputs, and figure/table generation notes.
- Reviewed literature and summarized findings into structured reports, improving problem definition and method selection.
```

Gap plan:

- create one business-style dashboard from public data
- write an insight report with clear assumptions

## Case 5: English / International Trade to Cross-Border E-Commerce

Candidate:

- English or international trade major
- no platform internship
- wants cross-border e-commerce operation

Positioning:

```text
Cross-border e-commerce entry candidate with English communication, product research, listing-copy, and market-comparison potential.
```

Resume strategy:

- emphasize English writing, product understanding, market research, Excel/data awareness
- avoid claiming platform operation if only studied it
- add listing portfolio

Example bullets:

```text
- Compared product listings across cross-border platforms, summarizing title structure, selling points, review concerns, and pricing signals.
- Rewrote English product descriptions for three sample SKUs, focusing on keyword clarity, buyer pain points, and concise feature-benefit expression.
- Completed course work involving international trade process, customer communication, and document preparation.
```

Gap plan:

- write 3 product listing samples
- create a competitor comparison table
- learn one platform's basic listing and ad vocabulary

## Case 6: Ordinary Resume to China Campus One-Page

Problem:

- too much self-evaluation
- scattered activities
- no clear target role
- long paragraphs

Fix:

```text
Name / contact
Target role
Education
Skills
2 strongest projects
1 internship or campus operation section
Awards/certificates
```

Rewrite principles:

- make the first half of the page carry the strongest evidence
- turn paragraphs into 2-3 compact bullets
- remove generic personality claims
- keep one accent color or none
