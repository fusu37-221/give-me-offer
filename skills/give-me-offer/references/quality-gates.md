# Quality Gates

Use this file before finalizing a resume, role strategy, or application package.

## Gate 1: Truthfulness

Pass if:

- no invented education, company, award, certificate, publication, role, dates, users, revenue, or production launch
- uncertain metrics are marked for confirmation
- ownership language matches actual contribution

## Gate 2: Role Fit

Pass if:

- target role is explicit
- top half of the resume contains matching evidence
- JD keywords appear naturally
- weakly related content is compressed or removed

## Gate 3: Scanability

Pass if:

- 3-second scan shows target role and identity
- 10-second scan shows strongest proof
- section headings are conventional
- bullets are compact
- one-page default is respected when appropriate

## Gate 4: Parser Friendliness

Pass if:

- no image-only text
- no heavy tables for core content
- no text boxes for key information
- contact details are not hidden in header/footer only
- source text can be copied cleanly

## Gate 5: Interview Defense

Pass if:

- strongest bullets have answer outlines
- risky AI/technical claims are downgraded or supported
- numbers can be explained
- candidate can explain the transition story

## Gate 6: Proof Plan

Pass if:

- weak resume has a short proof-pack plan
- portfolio task is realistic in 1 week
- proof aligns with target role
- resume wording after proof is defined

## Final Output Checklist

```text
Resume ready:
Role strategy ready:
Risky claims:
Questions for user:
Proof-pack next steps:
Interview-defense notes:
Application channels:
```
