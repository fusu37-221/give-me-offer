# Evidence Boundary

Use this file whenever resume enhancement may blur into exaggeration.

## Claim Levels

### Green: strengthen directly

Use confident language when the candidate can explain and support the claim.

### Yellow: translate conservatively

Use role language, but keep ownership and maturity realistic.

Examples:

- A student club event can become event planning, user communication, copywriting, registration funnel support, or cross-team coordination.
- A course project can become project experience, not company production experience.
- A full-stack developer who did some frontend can emphasize frontend modules, not claim frontend lead ownership unless true.
- A research assistant can emphasize literature review, data cleaning, experiment tracking, or technical writing according to actual work.

### Orange: confirm before use

Mark these as requiring user confirmation:

- exact numbers: users, revenue, conversion, latency, traffic, downloads, ranking
- team size and personal ownership
- whether a project was deployed, used, accepted, published, or awarded
- whether a tool or framework was used personally or only observed
- degree of independence
- employer/client names when sensitive

### Red: never generate

Do not invent or imply:

- degree, school, major, GPA, graduation status
- employer, internship, role title, work dates
- certificate, award, scholarship, publication, patent
- open-source contribution, GitHub stars, production launch
- team size, budget, revenue, user count, performance improvement
- language proficiency tests, visa/work authorization
- references, recommendations, background-check-safe facts

## Safe Enhancement Patterns

Prefer:

- "Built a course project that simulated..."
- "Participated in..."
- "Supported..."
- "Implemented the frontend pages for..."
- "Used X to complete..."
- "Gained hands-on familiarity with..."
- "Can explain the workflow of..."

Avoid unsupported:

- "Led the entire project..."
- "Independently delivered an enterprise system..."
- "Improved efficiency by 60%..."
- "Served thousands of users..."
- "Deep expertise in..."

## Interview Defensibility Test

Before finalizing any bullet, ask:

1. Can the candidate explain what they did in two minutes?
2. Can they answer follow-up questions about tools and tradeoffs?
3. Is the claim consistent with their seniority?
4. Would a background check, transcript, certificate check, or technical interview expose it as false?

If the answer is risky, downgrade the wording.
