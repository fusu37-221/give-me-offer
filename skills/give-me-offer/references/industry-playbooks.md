# Industry Playbooks

Use this file for common student, graduate, and career-switch routes. Treat these as starting patterns, not fixed templates.

## CS / Software Routes

### Full-Stack to Frontend

Positioning:

- "Frontend-focused engineer with full-stack collaboration experience."
- Emphasize UI modules, componentization, state management, routing, design implementation, API integration, and frontend quality.

Resume sections to foreground:

- frontend projects
- component abstractions
- page performance
- user-facing modules
- collaboration with backend/product/design

One-week proof:

- rebuild one existing project frontend with polished interactions
- document components and screenshots
- add responsive and accessibility checks

Interview explanation:

```text
My full-stack background helps me understand API contracts and product flow, but I want to specialize in frontend because I enjoy user-facing implementation, interaction detail, and engineering reusable UI.
```

### Full-Stack to AI Agent / AI Application

Positioning:

- "Application developer moving toward AI workflow automation and agentic product implementation."
- Emphasize API orchestration, tool calling, workflow state, data pipelines, RAG prototypes, prompt templates, evaluation logs, and business process automation when real.

One-week proof:

- build a small agent workflow with a narrow use case
- include architecture, tool list, failure cases, and evaluation examples
- avoid claiming research-level LLM expertise

### Backend to Data / AI Application

Positioning:

- Emphasize data modeling, APIs, ETL-like work, database queries, logs, analytics endpoints, and system reliability.

Gap to close:

- Python data stack, SQL analysis, basic ML/LLM application patterns, dashboard or notebook artifact.

## Product / Operations Routes

### Non-CS to Product Assistant

Positioning:

- "User-oriented candidate with structured analysis, communication, and requirement documentation experience."

Transferable assets:

- research reports
- student organization planning
- user interviews
- competitor analysis
- course projects with business logic
- written documents and presentations

One-week proof:

- one PRD
- one competitive analysis
- one user journey map

### Operations / New Media / E-Commerce

Positioning:

- Emphasize content planning, user growth, community management, event execution, data review, copywriting, and campaign iteration.

Safe translation:

- campus event -> event operation
- club account -> content operation
- volunteer recruitment -> user acquisition and communication
- questionnaire -> user research and data summary

One-week proof:

- 7-day content plan
- one account teardown
- one campaign review table

## Business / Cross-Border Routes

Emphasize:

- create a dashboard from public data

## Research / Graduate Routes

### Graduate Research to Industry R&D

Emphasize:

- research question framing
- literature review
- experiment design
- data analysis
- reproducibility
- technical writing

Avoid:

- overstating independent contribution if the project was supervisor-led
- converting papers into product impact unless there was real adoption

### Graduate Research to Data / Product

Translate:

- literature review -> information synthesis
- experiment tracking -> structured problem solving
- data processing -> analytical execution
- thesis writing -> documentation and stakeholder communication

One-week proof:

- notebook or report turning research data into an applied insight
- product memo based on research domain knowledge

## Traditional Engineering Routes

### Mechanical / Automation to Product / Implementation / IoT

Emphasize:

- systems thinking
- process understanding
- hardware/software interface awareness
- CAD or simulation if relevant
- on-site coordination
- documentation

Proof ideas:

- teardown of an industrial or IoT product
- implementation checklist
- workflow diagram for a real process
