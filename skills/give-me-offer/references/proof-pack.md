# Proof Pack

Use this file when a candidate's resume lacks evidence, when a career switch needs support, or when a role requires portfolio-style proof.

## Purpose

Many candidates do not need more decoration; they need proof. A proof pack gives the resume something defensible to point to.

## Proof Pack Types

### Software / Frontend

Useful proof:

- GitHub repository with clean README
- screenshots or short demo video
- component structure notes
- API integration examples
- responsive design screenshots
- bug/performance notes if real

One-week project:

```text
Rebuild one existing management-system frontend with polished pages, reusable components, form validation, loading/error states, and a README showing screenshots and structure.
```

### AI Agent / AI Application

Useful proof:

- narrow agent workflow
- architecture diagram
- prompt/tool/RAG notes
- sample inputs and outputs
- failure cases and limitations
- simple evaluation log

One-week project:

```text
Build a document Q&A or resume-screening assistant with file input, retrieval or structured context, tool/function flow if available, answer examples, and a limitations section.
```

### Product Assistant

Useful proof:

- PRD
- user journey map
- competitive analysis
- feature priority table
- interview/questionnaire summary
- prototype screenshots or wireframes

One-week project:

```text
Choose a familiar app feature, write a concise PRD, define users and scenarios, compare 3 competitors, and create a simple prototype or wireframe.
```

### Operations / New Media / E-Commerce

Useful proof:

- 7-day content calendar
- account teardown
- campaign review
- user persona
- conversion funnel notes
- product listing rewrite

One-week project:

```text
Analyze one real account/shop, summarize audience, content/product positioning, conversion path, and produce a 7-day execution plan.
```

### Data Analyst

Useful proof:

- notebook or Excel workbook
- cleaned dataset
- dashboard
- insight report
- data dictionary
- assumptions and limitations

One-week project:

```text
Use a public dataset to create a cleaned table, 3-5 charts, and a short business-style insight report.
```

### Research to Industry

Useful proof:

- research summary memo
- reproducible analysis file
- figure/table generation notes
- literature map
- method explanation
- industry application note

One-week project:

```text
Turn one thesis/research topic into a 1-2 page industry-facing memo explaining the problem, method, result, limitation, and possible application.
```

## Proof Pack Recommendation Format

```text
Current resume gap:
Best proof to add:
Why it helps:
One-week task:
Resume bullet after completion:
Interview talking points:
Risk if not completed:
```

## Rules

- Do not recommend fake proof.
- Prefer small complete projects over large unfinished ideas.
- A proof pack should be explainable in an interview.
- Keep proof aligned with the target JD.
- Add limitations; honest limitations increase credibility.
