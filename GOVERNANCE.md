# 治理说明

`give-me-offer` 作为一个实用型开源 Skill 维护。

## 维护者职责

- 保持 Skill 对真实求职者有用
- 审查内容是否符合“真实美化，不造假”的原则
- 维护岗位路线和市场热点，但不盲目追逐短期 buzzword
- 优先接受小而清晰、可验证的改动

## 决策原则

1. 候选人安全和可信度优先于激进包装
2. 真实证据优先于漂亮话
3. 长期能力优先于短期热点
4. 清晰流程优先于堆砌提示词

## 版本策略

早期使用草稿版本号，例如 `0.1.0-draft`。当 Skill 结构、references、脚本和模板稳定后，再采用更正式的语义化版本。
